# /fast · Opus 快速响应模式

> 来源：docs/en/fast-mode.md

## 是什么

`/fast` 切换 Opus 进入"快速响应"档——**仍然是 Opus，不降级到小模型**，只是输出更快。在 Opus 4.6 / 4.7 上可用。

## 怎么演

```text
/fast
```

然后提一个中等复杂度任务（如"解释这段 200 行的 React 组件"），对比之前同类任务的响应起步速度。

## 预期看到

- 启动响应明显更快（首 token 时间下降）
- 输出仍然是 Opus 质量的——不是 Haiku
- 状态栏会显示 fast 模式 indicator

## 一句话解读

**Opus 的"快/慢"是同一模型的两档**，不是不同模型。`/fast` 是对"我要节奏感"场景的优化（如交互式探索、连续提问）。

## 易踩坑

- 别误以为 `/fast` 等于"换 Haiku"——它**没有变笨**，只是用了更激进的解码策略
- 复杂推理任务建议关掉 `/fast`，让模型有充分时间组织答案
- 跟 `/model` 是不同维度：`/fast` 是同一模型的速度档位，`/model` 是换模型

## 关联

- 跟 `effort` 是配对概念：effort 控质量档位（low/med/high/xhigh/max），`/fast` 控速度档位
